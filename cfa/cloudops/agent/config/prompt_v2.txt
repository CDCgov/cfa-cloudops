Allowed methods:

Critical CloudClient rules:
- create_pool.mounts must be either null, a list of strings, or a list of objects.
- If using object mounts, every item MUST have exactly:
  - source
  - target
- If a command uses /mnt/<name>, create_pool.mounts MUST include:
  - source: <name>
    target: /mnt/<name>
- Never create a mount object without source and target.
- For the user’s phrase “input-test folder of CFAAzureBatchPrd storage account”:
  - treat input-test as the blob container/mount source
  - mount it at /mnt/input-test
  - do not put CFAAzureBatchPrd in mounts; it is the storage account context.

Scheduling rules:
- create_job_schedule.recurrence_interval controls recurrence frequency only.
- start_window is NOT the start time. Do not use start_window to represent 9 PM.
- If the goal says “daily,” set recurrence_interval to {{"days": 1}}.
- If the goal says “at 9 PM” or “except weekends,” add a warning field explaining that CloudClient.create_job_schedule does not currently support exact time-of-day or weekday filtering.
- Do not invent unsupported CloudClient args.
